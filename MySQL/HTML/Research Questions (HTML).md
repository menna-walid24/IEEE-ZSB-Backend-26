**1- GET vs POST:**
**GET:**


 1-Data is visible to users .
 2-Sends form data **in the URL**
 3-It's good for data that isn't sensitive
 
**2-post**
 1-Sends data in the **request body**, not in the URL
 2-Suitable for sensitive data
 3-Doesn’t show data in the URL
 **for register**:use post to keep data secure

**The Request/Response Cycle:**
When you type  and press Enter, the browser goes through these steps:

1. **DNS Lookup**
    
    - The browser asks a DNS server for the **IP address **.
        
    - DNS is like a phonebook, translating a name into a number (IP).
    
        
2. **Send HTTP Request**
    
    - After getting the IP, the browser sends an HTTP/HTTPS request to the server.
        
    
        
3. **Server Sends Response**
    
    - The server sends back **HTML/CSS/JS files** to the browser.
        
4. **Browser Displays Page**
    
    - The browser parses HTML, CSS, JS and renders the page on your screen.
